Class: Connection

Connection

Connection represents a call. A Connection can be either incoming or outgoing. Connections are not created by themselves but as a result on an action on Device. For example to initiate an outgoing connection you call Device.connect(parameters) which instantiates and returns a new Connection. On the other hand when an incoming connection arrives and you have previously registered a callback for receiving incoming connection events by calling RestCommClient.Device.incoming(callback), you will be notified through that callback and be passed the new Connection object that you can use to control the connection.

When an incoming connection arrives it is considered 'pending' until it is either accepted with Connection.accept() or rejected with Connection.reject(). Once the connection is accepted the Connection transitions to 'open' state.

When an outgoing connection is created with Device.connect(parameters) it starts with state 'pending'. Once it starts ringing on the remote party it transitions to 'connecting'. When the remote party answers it, the Connection state transitions to 'open'.

Once an Connection (either incoming or outgoing) is established (i.e. 'open') media can start flowing over it. DTMF digits can be sent over to the remote party using Connection.sendDigits(digits). When done with the Connection you can disconnect it with Connection.disconnect().
